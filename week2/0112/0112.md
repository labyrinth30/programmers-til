노드는 자바스크립트 런타임 환경이에요. 기존 웹브라우저에서만 사용 가능했던 자바스크립트를 브라우저 외에서 사용할 수 있게 만들어줘요.
오늘날에는 주로 서버 사이드 개발을 위해 사용되고 있어요.

### Node.js의 특징
구글의 V8 자바스크립트 엔진을 사용하여 JavaScript 코드를 빠르게 실행해요.
싱글 스레드로 작동하면서 특정 작업을 수행하는 동안 다른 작업이 동시에 처리될 수 있고, 이벤트 루프 등을 통해 많은 요청의 비동기적으로 효율적인 처리를 수행해요.

### Node.js로 간단 웹서버 만들기
```js
let http = require('http');

function onRequest(request, response) {
    response.writeHead(200, {'Content-Type' : 'text/html'});
    response.write('Hello Node.js');
    response.end();
}

http.createServer(onRequest).listen(8888);
```

```js
let http = require('http');
```
http 모듈은 모듈은 노드에서 http 서버를 생성하고 요청을 처리하기 위해 제공되는 기본 모듈이에요.
```js
function onRequest(request, response) {}
```
onRequest라는 함수는 http 요청이 들어올 때마다 호출되는 함수에요.
request는 클라이언트로부터 들어온 http 요청에 대한 정보를 담고 있는 객체에요.
response는 서버가 클라이언트에 응답을 보낼 때 사용하는 객체에요.
```js
response.writeHead(200, {'Content-Type' : 'text/html'});
```
응답 헤더를 설정할 수 있어요.
여기서 200은 성공를 나타내며, Content-Type header는 응답의 내용이 HTML 형식임을 지정해요.

```js
response.write('Hello World');
```
응답 본문에 Hello World라는 텍스트를 넣었어요.
```js
response.end();
```
이 명령을 호출해야 클라이언트가 응답을 받을 수 있어요.
```js
http.createServer(onRequest).listen(8888);
```
http 모듈의 createServer 메서드를 사용하여 서버를 생성하고 클라이언트에서 요청이 들어올 때마다 onRequest 함수를 실행해요.
서버를 8888번 포트에서 대기하도록 설정했기에 서버가 시작되면 localhost:8888에서 접근할 수 있게 돼요.

## 간단한 실습을 해봤어요.
![](https://velog.velcdn.com/images/ayeon0/post/4ae7b56d-b56b-4ad3-9164-e982a05b3f8d/image.png)

`index.js` - 서버 초기화 파일
서버를 시작하고 필요한 모듈을 불러와요.
`server.js`의 `start` 함수에 포트 번호, `route` 함수,`handle` 객체를 인수로 전달하여 서버를 초기화해요.
- index.js는 서버를 실행할 때 가장 먼저 호출되며, server.js의 start 함수로 제어를 넘겨요

![](https://velog.velcdn.com/images/ayeon0/post/4dd999ad-cb7c-4a0a-b51a-120dd36c5b5c/image.png)

`server.js` - 서버 구성 파일
http 서버를 생성하고 클라이언트의 요청을 처리해요. `route` 함수와 `handle` 객체를 사용하여 요청 URL을 기반으로 라우팅을 수행해요
- 클라이언트 요청이 들어오면 URL 경로를 추출하고, 해당 경로를 route 함수에 전달하여 콘솔에 경로를 출력하게 해요. 이후 handle 객체를 사용하여 경로에 맞는 핸들러 함수를 호출해 요청을 처리하게 돼요.
  ![](https://velog.velcdn.com/images/ayeon0/post/7c09d0f2-3747-4f9c-a2fa-0c8e067a313c/image.png)

`routes.js` - 라우팅 로직 파일
서버가 요청된 URL 경로를 콘솔에 출력해요. URL이 한글일 경우 인코딩된 URL을 디코딩하여 콘솔에 출력할 수 있도록 처리합니다.
- server.js에서 전달된 URL 경로를 decodedPathname으로 디코딩하고, handle 객체에 해당 경로가 있는지 확인해요. 이때 일치하는 핸들러가 없을 경우 콘솔에 `No handler found` 메시지를 출력하게 돼요

![](https://velog.velcdn.com/images/ayeon0/post/ed7fbd39-caf8-4ef1-b870-da21648aaf60/image.png)

`requestHandler.js` - 요청 처리 로직 파일
각 URL 경로에 맞는 핸들러 함수들을 정의하고, handle 객체에 경로별로 매핑해요. 요청된 경로에 맞는 함수를 통해 클라이언트에 응답을 보내요.
- handle 객체는 `server.js`에서 사용되어 요청된 URL에 따라 매핑된 핸들러 함수가 실행돼요.
  각 핸들러 함수는 요청 URL을 출력하고, 필요한 경우 특정 HTML 페이지를 응답으로 전송하거나 데이터베이스 작업을 수행하게 돼요.
